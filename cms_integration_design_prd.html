# Product Requirements Document: Premium Real Estate CMS Integration ## 1. Project Overview A high-end, editorial-style real estate frontend designed for seamless integration with a headless CMS (e.g., Strapi, Contentful, or Sanity). The UI is built using reusable components that consume dynamic data. ## 2. Design Philosophy - **Aesthetic:** Minimalist, high-end, editorial. - **Palette:** Off-white, warm gray, charcoal, beige. - **Typography:** Sophisticated serif for headings, clean sans-serif for body. - **UX:** Emphasis on high-quality imagery, whitespace, and smooth transitions. ## 3. CMS / Admin Structure Proposal ### Collections / Models #### A. Properties *Core model for all real estate listings.* - **Title:** String (e.g., "Villa Lumière") - **Slug:** Unique String (for URL generation) - **Status:** Enum (Available, Sold, Pending) - **Operation:** Enum (Sale, Rent) - **Price:** Number/String (e.g., "$4,500,000") - **Location:** String (e.g., "Saint-Tropez, France") - **Description:** Rich Text / Markdown - **Features:** Object (Bedrooms: Int, Bathrooms: Int, Area: Float/m2) - **Amenities:** Array of Strings (e.g., ["Infinity Pool", "Wine Cellar", "Gym"]) - **Media Gallery:** Array of Images (with Alt text and Captions) - **Map Coordinates:** Lat/Long or Address String - **Tags:** Array (e.g., "Waterfront", "Modernist", "Investment") - **Agent Reference:** Relation (Linked to Agents collection) #### B. Agents *Team member profiles.* - **Name:** String - **Role:** String (e.g., "Senior Consultant") - **Bio:** Rich Text - **Profile Image:** Image - **Contact Info:** Object (Email, Phone, WhatsApp) - **Social Links:** Array of Objects (Platform, URL) #### C. Testimonials *Customer social proof.* - **Client Name:** String - **Quote:** Text - **Property Referenced:** Relation (Optional link to Properties) - **Avatar:** Image (Optional) #### D. Homepage Content *Singleton or Page-specific model.* - **Hero Title:** String - **Hero Media:** Image or Video URL - **Intro Text:** Text - **Value Propositions:** Array of Objects (Icon, Title, Description) ## 4. Component Architecture - **PropertyCard:** Consumes `property` object (thumbnail, title, price, stats). - **PropertyGallery:** Handles array of images with lightbox/thumbnail navigation. - **AgentCard:** Displays agent profile and contact quick-links. - **FilterSystem:** State-driven UI for querying the CMS API. ## 5. Technical Implementation Notes - **Dynamic Routing:** Use slugs (e.g., `/properties/{{property.slug}}`) for SEO-friendly detail pages. - **Media System:** Responsive image handling (Srcset) for high-res property photography. - **Data Fetching:** Standardized hooks/functions to map CMS JSON responses to component props.
